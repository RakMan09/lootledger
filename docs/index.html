🛡️ Why it can't dupe

    
    
      
  • Double-entry ledger. Value never appears; every change writes balanced
        postings that sum to zero, so gold can't be created or destroyed by accident.
    • 
      
  • Idempotency keys. Each request carries a unique key; the first execution is
        recorded and every duplicate replays the same stored response. In the real system a Postgres
        UNIQUE constraint makes this atomic and race-proof.
    • 
      
  • Trades are a saga. Escrow each side → cross → complete, each step idempotent;
        a crash resumes to completion or fully compensates — value is never stranded.
    • 
      
  • Exactly-once loot. Kafka events are keyed by loot id, so redelivery is a no-op.
    • 
      
  • Reconciliation. A scheduled job re-proves conservation and flags any drift.
